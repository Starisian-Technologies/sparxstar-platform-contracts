# Security Policy

## Supported Scope

This repository contains authoritative interface contracts, governance metadata, and documentation for the SPARXSTAR platform. Security reports are welcome for:

- contract definitions that could enable insecure integrations
- repository automation or workflow weaknesses
- documentation that materially misstates security requirements
- supply-chain or dependency risks in repository tooling

## Reporting a Vulnerability

Do not open public issues for security vulnerabilities.

Use one of these private channels instead:

1. Open a private GitHub security advisory for this repository.
2. Contact the repository owner directly if you already have an established private channel with Starisian Technologies.

Include:

- a clear description of the issue
- affected files or workflows
- reproduction steps or proof of concept
- impact assessment
- suggested remediation, if known

## Response Expectations

- Initial triage target: 5 business days
- Status update target: 10 business days
- Remediation timing depends on severity, exploitability, and sync requirements with source service repositories

## Disclosure Guidance

- Please keep details private until Starisian Technologies confirms remediation or accepts the risk.
- If the issue originates in a source service repository, it may be fixed there first and then synchronized into this repository.

## Out of Scope

- requests for support using the contracts
- product feature requests
- issues caused solely by downstream implementations that do not follow these contracts
