# Schemas

Shared JSON/YAML schemas. This is the repo's contract surface — the source
of truth `ROLE.md` and `docs/dictionary-tech-spec.md` point to under
"Contracts produced." Anything here is synced to the platform contracts repo
(`Starisian-Technologies/sparxstar-platform-contracts`, path
`Contracts/IAMC/Dictionary/`) by `.github/workflows/sync-contracts.yml` on
every push to `main` that touches this directory.

## Files

- `dictionary-openapi.yaml` — OpenAPI 3.0 spec for the public REST API at
  `sparxstar/v1/dictionary`. Covers every consumer-facing endpoint (lookup,
  search, wordlist, languages, domains, game-set, word-of-day, pronounce,
  page-token, spell) with request/response shapes and error format. Models
  both pre-shared credential schemes (`X-Page-Token`, `X-Api-Key`) where
  they apply — `page-token` and `spell` are documented as unauthenticated
  (`security: []`), matching their actual public/rate-limited handlers.
  Validate with `npx @redocly/cli lint schemas/dictionary-openapi.yaml`
  before committing changes.

`POST /progress/sync` is intentionally excluded — it's deprecated, frozen,
and its own docblock says not to document it publicly (see
`Sparxstar3IAtlasDictionaryRestApi::register_rest_routes()`).

## Keeping this in sync with the code

This file is generated by hand from the actual endpoint handlers in
`src/api/`, not the other way around. When an endpoint's request/response
shape changes, update this spec in the same PR — a stale contract is worse
than no contract for downstream consumers who trust it.
